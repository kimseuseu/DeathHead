# DeathHead

R.E.P.O. 스타일 사망 연출 + 아이템 봉인/회수 시스템 Paper 플러그인

## 기능

### 사망 연출
- 항공샷 카메라 (관전모드 전환 + 상공 텔레포트)
- 폭죽, 소울 파티클, 블라인드니스 페이드인
- "☠ YOU DIED" 타이틀 표시

### 아이템 봉인
- 사망 시 인벤토리의 10~15%를 플레이어 머리에 봉인하여 드롭
- 봉인할 아이템이 없으면 기능 없는 빈 머리만 드롭 (발광 + 이름 표시)
- 열쇠, 사망 방지권, 봉인된 머리는 유실 대상에서 제외
- 채팅에 유실된 아이템 목록 표시 (호버 시 아이템 정보 확인 가능, 한글명)
- 사망 좌표 표시

### 머리 시스템
- 드롭/설치된 머리 위에 패킷 기반 TextDisplay로 소유자 이름 + 부패 카운트다운 표시
- TextDisplay 포맷을 messages.yml에서 드롭/설치 별도 커스텀 가능
- 발광 효과로 블록 뒤에서도 머리 확인 가능
- 인벤토리에서 버린 머리도 발광 + TextDisplay 자동 적용
- 머리를 설치 후 열쇠로 우클릭하면 봉인 해제
- 인벤토리에서 머리 Shift+우클릭으로 봉인 아이템 미리보기
- 인벤토리 내 머리 로어에 남은 시간 실시간 표시 (1초 단위)
- 열쇠/방지권은 블록 설치 불가

### 부패 시스템
- 일정 시간(기본 60분) 후 머리가 부패
- 부패 시 로어, 발광, TextDisplay, 스케줄러 모두 제거 → 기능 없는 일반 머리로 전환
- 드롭 상태, 설치된 블록, 인벤토리 내 모두 동일하게 적용

### 보호
- 사망 방지권 소지 시 아이템 유실 없음 (1개 소모)
- 폭발(크리퍼/TNT) 파괴 시 PDC 데이터 보존 + 클리어랙 보호 유지
- 설치 후 파괴해도 deathhead 태그, 발광, 무한 수명, TextDisplay 재적용
- 공허/용암/선인장은 데미지 허용
- 바닥 아이템 자연 소멸 방지 + 더미 패신저로 클리어랙 대응
- 서버 재시작 시 드롭된 머리 TextDisplay + 타이머 자동 복구

### 최적화
- 패킷 기반 TextDisplay (ProtocolLib) — 실제 엔티티 없음, 서버 틱 부하 없음
- NMS 리플렉션으로 Vector3f translation 적용 (마운트 + Y 오프셋)
- 인벤토리 열림 감지 (WINDOW_CLICK 패킷) — 열고 있을 때만 로어 타이머 갱신
- 거리 기반 가시성 관리 (48블록)
- 로어 텍스트 변경 없으면 setItemMeta 호출 스킵

### 기타
- 열쇠 크래프팅 레시피
- 접속 시 관전모드 복구 처리
- 열쇠/방지권은 디스플레이 이름으로 식별 (PDC 미사용)

## 명령어

| 명령어 | 설명 | 권한 |
|--------|------|------|
| `/dh` | 도움말 | - |
| `/dh key [플레이어] [개수]` | 열쇠 지급 | `deathhead.admin` |
| `/dh ticket [플레이어] [개수]` | 사망 방지권 지급 | `deathhead.admin` |
| `/dh reload` | 설정 리로드 | `deathhead.admin` |

## 설정 파일

| 파일 | 설명 |
|------|------|
| `config.yml` | 패널티 비율, 부패 시간, 쿨다운 등 |
| `messages.yml` | 채팅 메시지 + TextDisplay 포맷 커스텀 |
| `items.yml` | 열쇠/방지권 아이템 이름, 로어, 데이터 커스텀 |

## 요구사항

- Paper 1.21.11+
- Java 21+
- ProtocolLib (필수)

## 빌드

```bash
mvn clean package
```

`target/DeathHead-2.5.0.jar` 생성 → `plugins/` 폴더에 배치
